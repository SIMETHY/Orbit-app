# ORBIT — Enterprise Asset & Calibration Intelligence Platform

A full-stack prototype: **FastAPI + SQLite backend** and a **React (Vite) frontend**,
covering asset search, calibration tracking, reusable material cataloging, movement
logging, procurement duplicate-checking, and a rule-based AI insight feed.

```
orbit-app/
  backend/    FastAPI + SQLAlchemy + SQLite API
  frontend/   React + Vite dashboard (dark "asset tag" themed UI)
```

## 1. Run the backend

```bash
cd backend
python -m venv venv && source venv/bin/activate    # optional but recommended
pip install -r requirements.txt

python -m app.seed          # creates orbit.db and seeds demo data
uvicorn app.main:app --reload --port 8000
```

Backend now runs at `http://127.0.0.1:8000`. Check it's alive:
`curl http://127.0.0.1:8000/api/health` → `{"status":"ok","service":"orbit-api"}`

Interactive API docs (auto-generated by FastAPI) are at
`http://127.0.0.1:8000/docs`.

## 2. Run the frontend

In a second terminal:

```bash
cd frontend
npm install
npm run dev
```

Open `http://localhost:5173`. The Vite dev server proxies `/api/*` requests
to the backend on port 8000 (see `frontend/vite.config.js`), so no extra
configuration is needed.

## 3. What's wired up end-to-end

| Screen | Backend endpoint(s) |
|---|---|
| Command Dashboard | `GET /api/analytics/dashboard`, `GET /api/insights` |
| Asset Explorer (search/filter, reserve) | `GET /api/assets`, `PATCH /api/assets/{id}/status`, `GET /api/labs`, `GET /api/business-units` |
| Calibration Center | `GET /api/calibration/register`, `GET /api/calibration/summary` |
| Reusable Materials Bank | `GET /api/reuse-items` |
| Movement & Custody Log | `GET /api/movements`, `POST /api/movements` |
| Procurement Duplicate-Check | `GET /api/procurement/check?q=...` |
| AI Insight Feed | `GET /api/insights` |

All of it is backed by a real SQLite database (`backend/orbit.db`, created on
first run) — not hardcoded frontend data. Reserving an asset or checking
procurement actually reads/writes the database.

## 4. Swapping in a "real" database

The backend defaults to SQLite for zero-setup local demo use. To point it at
Postgres instead, just set an env var — no code changes required:

```bash
export ORBIT_DATABASE_URL="postgresql://user:password@host:5432/orbit"
```

## 5. What's a stand-in vs. production-ready

See the in-app **Module 08 — Platform Blueprint** tab for a full breakdown,
but in short: search/matching uses SQL `ILIKE` (swap for embeddings in
production), the AI insight engine is rule-based over live data (swap for an
LLM call reasoning over the same tables), there's no auth yet, and tagging is
seeded rather than scanned from physical QR/RFID tags.

## 6. Extending the seed data

`backend/app/seed.py` contains all demo data (labs, business units, assets,
movements, reuse items, compliance trend) as plain Python lists — edit those
and re-run `python -m app.seed` to reset the database with new data.
